# 标题规则（WeChat + XHS 通用）

核心逻辑：标题要制造"缺口"，不要做"说明书"。

读者点进来不是因为看懂了标题，而是因为标题让他有一个没被填满的好奇心。

## 好标题的三种类型

1. 具体数字 + 意外结果
- 用真实数字造成落差感，读者会想"然后呢"。
- 示例：
  - 《我花了七八十块 token，一个功能都没跑通》
  - 《ChatGPT 五个账号花了我不到五块，Kimi 花了我 199》

2. 个人立场 + 悬念
- 有明确立场，但不说完，让读者想知道为什么。
- 示例：
  - 《满屏 OpenClaw，我下场试了，建议你先别》
  - 《我在 AI IDE 里备考，不是在学编程》

3. 反常识 / 让人愣一下的句子
- 完全看不出文章讲什么，但就是想点进去。
- 示例：
  - 《我对不起奥特曼》
  - 《我把"会失忆的老师"变成了一个文件夹》

## 禁止使用的标题结构

- "XXX：我不是 A，我是 B"（解释型）
- "如果你……，这篇是给你的"（定向型）
- "XXX 的 N 个方法/技巧/误区"（清单型）
- 副标题比主标题更长

## 执行要求

- 每次写作前，必须给出 3 个候选标题，让用户选。
- 3 个候选标题要覆盖上述三种类型各一个，不要三个都是同一种逻辑。
