# 去 AI 味写作流程（工作草稿 -> 人话成稿）

## 目标

在不增加虚构信息的前提下，让成稿更像真人表达：口语化、有节奏、具体、直接。

## 三阶段流程

### 阶段 1：先写“人话草稿”（不使用 AI）

30 分钟定时，一口气写完，先不管结构。

**写作要求**
- 不管语法和结构
- 用大白话，像跟朋友聊天
- 写具体事，不要抽象观点

**写完 4 个部分（总计 300–500 字）**
1. 为什么做：问题是什么，具体有多难受，为什么想到用 AI  
2. 怎么做的：用的工具、3–5 步流程、关键设计  
3. 遇到什么坑：失败点、尝试过的方法、最后怎么解决  
4. 效果如何：具体数据，哪里好/不好，有什么局限  

### 阶段 2：用 AI 做“去 AI 味润色”

**固定提示词模板（直接使用）**

```
我写了一篇关于AI工作流的文章草稿，需要你帮我润色。

【重要要求】
你的任务是让文章更清晰、更有条理，但必须保持我的语气和风格。

【节奏要求】
1. 长短句结合：不要全是短句，也不要全是长句  
   - 背景铺垫：可用长句（20–30字）  
   - 方法介绍：可用长句，但要分段  
   - 重点强调：用短句（10–15字）  
   - 举例说明：可用长句  
2. 适当使用连接词：  
   - 可以用：所以、但是、因为、比如、结果  
   - 避免用：然而、因此、基于、催生、驱动  
3. 保持呼吸感：  
   - 每2–3句长句后，加一个短句  
   - 每一大段（5–7句）后，空一行  
   - 不要连续10句都是短句  

【语言要求】
1. 保持口语化：像聊天一样，不要太书面  
2. 保持真实：不要添加我没说过的内容  
3. 加入具体例子：若出现抽象概念，提醒我补具体例子  

【你可以做的】
1. 把连续的短句合并成长句（用逗号、冒号连接）  
2. 把一个长句拆成2–3个短句（如果超过40字）  
3. 调整段落，让节奏有变化  
4. 修正错别字和语病  

【你不能做的】
1. 改变我的用词习惯  
2. 添加我没说过的案例  
3. 把我的大白话改成文学语言  
4. 删掉我想强调的重复内容  

【检验标准】
- 读出声，看是否顺畅、有节奏  
- 每段应该有2–3个长句 + 1–2个短句  
- 不应该连续出现超过5个短句  

【你可以做的】
1. 调整段落顺序，让逻辑更清晰
2. 给每个部分加小标题（简单直接）
3. 把啰嗦的地方改简洁
4. 修正错别字和语病

【你不能做的】
1. 改变我的用词习惯
2. 添加我没说过的案例或数据
3. 把大白话改成文学语言
4. 加上我没写的内容

【检验标准】
我读完每一句都觉得“这就是我想说的”，不是“AI帮我说得更好了”。

这是我的草稿：

[粘贴你的草稿]

现在开始润色。润色完后，请在文章末尾列出：
1. 你做了哪些改动
2. 你有没有添加任何我原文没有的内容（如果有，请标注出来）
```

### 阶段 3：人工复检（关键）

**检查清单**
- 这句话是我会说的吗？不是就改回
- 这个词我平时会用吗？  
  - “颇为”→“挺”  
  - “催生”→“让我想到”  
  - “驱动”→“推动我/让我”  
- 例子准确吗？AI 加的就删  
- 有信息量吗？没信息量就删  
- 读出声顺不顺？别扭就改  

## AI 味自检表（快速检索）

| 特征 | 检查 | 处理 |
| --- | --- | --- |
| “然而/因此/基于/催生/驱动” | 全文搜索 | 改成口语连接词 |
| 长短句失衡 | 数每段长短句 | 调整节奏 |
| 连续短句过多 | 看是否>5句 | 合并或加长句 |
| 抽象多具体少 | 数具体例子 | 补具体细节 |
| 读起来没呼吸 | 读出声 | 分段、加短句 |

## 额外禁区（常见“宣传片口气”）

下面这些词/句式一出现，读者很容易出戏（更像 AI 或公关稿），优先改成大白话：

- “宏大的技术理想/愿景/使命” → 直接说当时你想解决的具体麻烦
- “催生了我的想法/驱动我开始探索” → “我就想：找个办法”“我当时直接去试了…”
- “最朴素的想法” → 删掉，直接把想法说出来
- “一系列长期困扰我的问题” → 列 2–4 个具体问题

## 快速版本（短提示词）

```
帮我把这段话改得更清晰，注意：
1. 长短句结合，有节奏
2. 允许用：所以/但是/因为/比如/结果
3. 避免用：然而/因此/基于/催生/驱动
4. 不加我没说过的内容
```
